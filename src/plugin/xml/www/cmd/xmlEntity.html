xmlEntity - Edit the table of named entities

xmlEntity entity = expr
xmlEntity
This command is part of the xml plugin. It is only available if you used the -lxml command-line flag or the plugin xml command.

Entities are stored in config.plugin.xml.entity — an object where the member key is an entity name and the member value is the text that the entity represents. However, the normal set command won't let you add members to an object. The only way to add a new entity to that object is via xmlEntity.

Note that the text is given as an expression. This means that if you want it to be literal text, you must enclose the text in quotes. You can also give a number, which will be interpreted as a Unicode codepoint for a single-characters string.

You can also invoke xmlEntity without any arguments to display the whole entity table.

Examples
xmlEntity
entitycodepointtext
&amp;U+00026&
&apos;U+00027'
&gt;U+0003e>
&lt;U+0003c<
&quot;U+00022"
 Dump the entity table. Initially it only contains the five basic entities.
xmlEntity ntilde="ñ" Add an entry to make &ntilde; be converted/displayed as ñ.
Notes
  • If the parseEntity option is turned on, then any <!ENTITY name=text> entities will also add entities to the table.
  • The XML parser also understands &#decimal; and &#xhex entities. These don't need to be in the table.
