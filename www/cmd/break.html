break - Exit a loop or switch statement

break
When used within a switch statement, it marks the end of a case; without break, control flow would continue into the next case.

When used within a while or for loop, it causes the loop to end immediately.

Examples
var i
for (i = 0 .. (test.scores.length)) { if (test.scores[i].name == "Mary") break;}
i 2 This declares a variable, and then uses it to loop over the subscripts of the test.scores array (part of the sample data). When it finds a score with the name "Mary", it stops scanning. The final "i" occurs after the loop, and is used to output the index where the record was found.

switch (3) {case 2: case 3: case 5: case 7: "prime";break; default: "non-prime";} "prime" Another ugly example due to lack of newlines; sorry about that.

The four case statements run together because there is no break between them, so they all do the "prime" thing. And then they hit break so they do not fall though to also say "non-prime".

Notes
  • The break statement always exits the innermost switch, while or for statement. If you have nested loops, for example, and use break in the inner loop then the outer loop will not terminate yet.
