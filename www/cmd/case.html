case - Defines a branch label within a switch statement

case literal:
This only works within a switch statement. The expression in the switch is compared to the literal in the case statement, and if they match then the code after the case statement is executed.

Note that the case statement ends with a colon, not a semicolon or end of input like most other commands.

Examples
switch ("Y") { case "Y": true; break; default: false; } true Because the expression in the switch is "Y", the case "Y" code is executed and it outputs true.
Notes
  • You'll generally want to use a break command at the end of a case's code. One big exception is that if you want multiple cases to share the same code, then you can just have a series of case statements all strung together, followed by some shared code and a break or return.
    switch (entry) {
    case "Yes":
    case "Oui":
    case "Sí":
    case "Ja": "Affirmative"; break;
    default: "Negative";
    }     This compares the enty variable to multiple possible affirmative words.
