explain - Describe a table's format

explain table
explain
explain ?
The explain command describes a table. If you don't specify a table via an expression, then it will use the default table and, for an added thrill, tell you where the default table is located. A third option is to pass just an question mark, in which case it'll tell you where the default table is located but not describe it.

Tables are arrays of objects. Each element (object) is a row of the table, and each member of the object is a column. The explain command collects statistics about the columns across all rows. The statistics kept are:

keyThe name of the member/column.
typeThe data type. This is described more in the typeOf() function's page.
widthDisplay width of the widest value for this column, expressed as characters. See the widthOf() function's page for details.
nullableWhether this column is missing from some rows, or is present with a null value.

Notes
  • This scans every element of every object in the table. For a large table, this can take some time.
  • This command is basically a convenient wrapper for the explain() function. MySQL has a handy "explain" command, and JsonCalc's explain command mimics it pretty well. Also, the command can locate and display the default table when invoked without any arguments; the explain() function can't do that.
