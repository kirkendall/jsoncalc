import - Load jx code from another file

import filename
Load jx code from another file. This can be used to split large scripts into multiple files, so you load one script and it uses import to load the rest. This can also be handy when running interactively if you decide that a written-but-not-loaded script might be handy for analyzing or converting the data.

The import command doesn't stop you from loading the same file twice. This is intentional. You might find a bug in a script, edit it in another window to correct the bug, and then use import to reload the corrected version.

If the filename you give doesn't include a filename extension, then ".jc" will be assumed.

Examples
import stats.jc This reads any variables or functions from stats.jc, so they're available for your script or interactive session.
Notes
  • I used the name "import" to make this command easier for JavaScript programmers to find, but are significant differences.
    • JavaScript can't load from files, only from HTTP/HTTPS URLs. jx can only load from files.
    • JavaScript has extra keywords to control which parts of the external script get loaded. jx just loads everything.
    • JavaScript requires quotes around the external script's name. jx doesn't, partly for convenience and partly to make import more closely resemble the file command for loading data files.
  • For security reasons, the script file's name can't start with "/" or contain "../".
  • There is also a plugin command which loads plugins. Plugins can be implemented as either a binary file containing native code, or a jx script, or a mixture of the two. Since both import and plugin can load jx scripts, there is a bit of overlap. The plugin command is more powerful, but it looks for plugins to load in separate directories. Use import for your scripts, and plugin for code contributed by other programmers.
