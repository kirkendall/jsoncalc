charCodeAt - Return the codepoint (number) of a character

charCodeAt( str:string, pos?:number|array ) :number|array
This returns the numeric code (e.g. Unicode or ASCII) of a character at a given position within a string. The first character in a string is at position 0.

If charCodeAt() is invoked with only a str string parameter (no position) then it returns the character code for the first character.

If invoked with a string str and number num, it skips num characters into the string and returns the code for the next character.

Unlike JavaScript, JsonCalc also allows you to pass an array arr of position numbers. When you do this, it returns a corresponding array of character codes. This can be handy because JsonCalc's fromCharCode() function also accepts an array of values, so one can undo the effects of the other.

Examples
"Steve".charCodeAt() 83 83 is the ASCII/Unicode codepoint for the uppercase letter "S".
"Steve".charCodeAt(1) 116 Passing 1 as the position causes charCodeAt() to skip over the "S", and return the code for "t".
"Steve".charCodeAt(1...3) [116,101,118] Here we're passing an array of positions, and getting an array of character codes back. The expression 1...3 returns the array [1,2,3].
Notes
  • Although some UTF-8 characters require more bytes than others, you can ignore that here. This function counts characters, not bytes.
