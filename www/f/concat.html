concat - concatenate arrays or stringsOne line summary

concat(arr, ...)
concat(str, ...)
Concat takes a variable number of arguments.

If all of the arguments are arrays, then it returns a new array containing all of the elements from the argument arrays.

If all of the arguments are strings (or numbers or booleans which are converted to strings), then it returns a new string containing the concatenation of all of those strings. null arguments are ignored.

Examples
concat("a", "beehive", "c", 3, null, true, "g") "abeehivec3trueg" The null is ignored, but the other arguments are all converted to strings and combined to form one long string.
var a=[1,2,3]; a.concat([4,5]) [1,2,3,4,5] a [1,2,3] Here we're using dot notation and doing an array concatenation. Note that the a variable is unchanged; concat() always returns a new array.
Notes
  • This function is common in SQL.
  • concat() differs from the join() function in that concat() expects all strings to join to be given as separate arguments, while join() expects either an array of strings, or (with SELECT, @ or @@ operators) single strings that are aggregated across rows of a table.
  • String concatenation can also be done via the + operator. Both JavaScript and SQL allow concat() to concatenate strings, so JsonCalc does too. Also, JsonCalc's + operator treats date/time/datetime/period strings specially but concat() does not, so that's sometimes useful.
  • You can also append an item to an array variable by using the []= appending form of the assignment operator.
