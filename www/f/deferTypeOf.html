deferTypeOf - Return the type of deferring used for a given array

deferTypeOf(arg:array):string
If an array is deferred, this will return the name of the type of deferring that is being used. If the argument is a non-deferred array, or not an array at all, then it returns null.

This rarely matters. Deferred arrays, regardless of their origin, are treated almost exactly like non-deferred arrays. Altering an array, for example by appending a new element to it, will cause the array to be converted to a non-deferred array which can take a noticeable amount of time and memory, so maybe in some very rare cases you might want to see if deferTypeOf() returns a non-null value.

Examples
deferTypeOf(1...1000000) "Ellipsis" The ... operator returns a deferred array of integers. This function returns "Ellipsis" to indicate that it is indeed a deferred array, and that specifically it was created by "...".
Notes
  • At present, the types of deferred arrays (i.e., the strings this function can return) are:
    "Blob" Binary data, presented as an array of byte values.
    "Ellipsis" An array of integers over a given range.
    "Dummy" Used internally by the json_by_expr() C function as an aide to memory management.
    "JSON" Large array in parsed data from a *.json file
  • Plugins can implement their own types of deferred array.
