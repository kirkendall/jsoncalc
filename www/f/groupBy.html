groupBy - Split a table into groups

groupBy(table, columns) groupBy(table, columns, totals)
This splits a table (array of objects) into an array of arrays of objects.

This is mostly to affect the scope of aggregate functions when used with the @ and @@ operators. For example, if you have a table of sales data which contains a "region" column (meaning each row/element/object has a "region" member) then you might want to group them by region so you could report the regional totals or averages. (Yes, this corresponds to the SQL "GROUP BY" clause.)

The table is simply an array of objects. Each element is a row, and then members of the rows' objects serve as columns.

The columns list is an array of strings, corresponding to the object member keys (column names). For the sake of convenience, if you just want to sort by a single column in ascending sequence, you can just pass its name as a string, instead of an array of strings.

This doesn't sort the items; it is assumed that all items to be grouped are already in consecutive rows. If that assumption isn't true, then you should sort then via orderBy() before calling groupBy(). You can use the same columns parameter for both. (groupBy() only compares for equality so it doesn't need true to denote descending sorts, but it is smart enough to skip over true if it appears in the columns argument.)

If you pass a totals parameter, then orderBy() will append an extra object to the end of the response array, containing a single member with first name from the columns parameter as the member's key (name), and the totals value as its value. Due to the way @ and @@ interact with aggregates, this causes any columns that use aggregate functions to apply those functions across all rows instead of splitting by the group. This makes it easy to get combined totals.

Examples
groupBy(test.scores, "run")@{run,avg(score)}
runavg(score)
120
230
331.6666666667
 The "test.scores" table is already sorted by "run" so we don't need to explicitly sort it. The @ operator outputs one record per group, in this case containing the run number and the average score within that run.
test.scores.orderBy("name").groupBy("name","OVERALL")@{name,avg(score)}
nameavg(score)
Mary35
Paul25
Peter21.6666666667
OVERALL27.2222222222
 This time we're fetching the averages by name, so we need to sort and group by name. Also, we added "OVERALL" as a totals label, so we got a row for the overall average too.
test.scores.groupBy("run")@@score<avg(score)?{run,name,score}
run namescore
1Peter10
2Peter25
3Peter30
3Paul25
 The difference between @ and @@ is that @ outputs one row per group, while @@ outputs one row per row. We're using @@ here to list the run, name, and score of anybody who got less than the average score for their run. Note that we got two rows from run 3.
Notes
  • The result of groupBy() is no longer a table. Instead, it is an array of tables, possibly with an oddball "totals" object added in. The @ and @@ operators work with this, and will reassemble the results as a table again.
