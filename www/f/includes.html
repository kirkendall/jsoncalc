includes - Test whether an array or string contains a value

includes(arr, elem)
includes(arr, elem, true)
includes(str, substr)
includes(str, substr, true)
The includes function searches through either an array or a string, and returns true if it contains a given value, or false otherwise.

When searching through an array, the second argument is compared to each element of the array. The comparison is similar to the === operator — you can search for complex values such as objects or nested arrays, not just numbers or strings.

When searching through a string, it looks for any instance where the second argument appears as a substring of the first string

When comparing strings (either as an element of an array, or as a substring within a string), comparisons are normally case-sensitive. You can append a true argument to make it be case-insensitive.

Examples
"I have a cat on my lap".includes("cat") true There is indeed a cat on my lap.
"I have a cat on my lap".includes("dog") false There word dog doesn't appear anywhere in the sentence.
"Once Upon a Time".includes("up") false "Once Upon a Time".includes("up", true) true Searching for "up" case-sensitively found nothing, but a case-insensitive search found it.
[1,2,3,"four","Five",3,2,1].includes(3) true There is indeed at least once instance of 3 in that array.
[1,2,3,"four","Five",3,2,1].includes("five") false [1,2,3,"four","Five",3,2,1].includes("five", true) true A case-sensitive search failed to find "five", but a case-insensitive search found it.
[{x:1,y:2},{x:3,y:4},{x:5,y:6}].includes({x:3,y:4}) true Here's an example of searching for a complex value. The object {x:3,y:4} does indeed appear within the array.
Notes
  • JavaScript has no case-insensitive option. Passing true as the third parameter is strictly a JsonCalc thing.
  • JavaScript's version of includes() can't search arrays for complex values.
