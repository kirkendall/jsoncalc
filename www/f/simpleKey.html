simpleKey - Transform a string to allow "loose" comparison

simpleKey(str:string):string
The simpleKey() function transforms a string to allow for easy comparison. The "case insensitive" member name lookup uses the same rules as this function, so experimenting with this function can give you a better feel for how loose lookup works.

The transformations performed include:

  • Everything before the first ":" is deleted.
  • Control characters are deleted.
  • Zero-width characters (diacritics, mostly) are deleted.
  • Precomposed diacritics are converted to their base letter.
  • Lowercase letters are converted to uppercase.
  • ß and Æ/æ are converted to SS and AE respectively.
  • "-" and "_" are removed, except at the beginning or end of the string.

Notes
  • Deleting everything before ":" is intended to let you access XML symbols without worrying about namespaces.
  • Deleting "-" helps with data from languages that allow hyphens as part of a name, such as XML and CSS.
  • Deleting "_" lets us avoid the snakes-vs-camels debate.
  • Converting to uppercase avoid a lot of frustration, for example whether an identifier member's key ends with "Id" or "ID".
  • Removing diacritics just saves time when typing.
  • The simplified string never requires more bytes than the original string. In Unicode, the ß, Æ, æ are represented by 2-byte sequences, so converting them to two 1-byte letters doesn't use any extra memory. (This doesn't really matter in JsonCalc scripts, but I thought it was interesting.)
