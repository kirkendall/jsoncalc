slice - Return part of an array or string

slice( subject:array|string, start:number, end?:number ) :array|string
The slice() function returns a new array filled with a portion of the given subject array, or a substring from a string.

If the start argument is 0 or more, then it indicates how many elements or characters to skip from the subject. If start is negative, then it counts from the end of subject.

If end is omitted, then the slice extends to the end of subject. Otherwise, if 0 or positive then it indicates where the slice ends in subject; i.e., it is the index of the first element of subject to omit from the slice, or the first character to omit. If it is negative, then it counts from the end of subject instead of the beginning.

Examples
(0...10).slice(3,5) [3,4] The (0...10) returns an array of integers from 0 to 10. The .slice(3,5) skips the first 3 elements, and returns the text elements up to but not including the 5th element.
(0...10).slice(-1) [10] This time we start copying from 1 element from the end of arr. There is no end parameter so the slice extends to the end of arr.
"Billiards".slice(3, 7) "liar" Just a quick example to demonstrate that it works on strings.
Notes
  • If the end point is before the start point, then an empty array or string is returned.
  • For strings, the main difference between slice and substr() is the way they indicate the end of the returned substring. slice() is passed the position to end at, while substr() is passed the desired length.
  • Why do we have JavaScript's slice() function but not splice()? It's because JsonCalc's expressions don't have side-effects. splice() is all side-effect.
