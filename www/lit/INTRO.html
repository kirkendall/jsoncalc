JsonCalc Data Types and Literals

Expressions in JsonCalc operate on JSON data, so JsonCalc's basic data types are the same as JSON's. Here we describe how those types are represented in JsonCalc expressions. (Spoiler: About the same as JavaScript.)

Name Description
null Used to represent no value, or an error.
boolean Used to represent Boolean values true and false.
number Digit strings used to represent numbers.
string Character strings.
date, time,
datetime,
period		 ISO 8601 dates and times.
array Array generators construct new arrays from other literals or expressions.
object Object generators construct new objects from other literals or expressions.
name Names/keys. Not a literal, but syntactically close and I need to dissuss them somewhere so...
$NAME Also not really a literal, this is one way to access environment variables.
regex Regular expressions aren't technically literals either since they aren't a valid JSON data type, but JsonCalc supports them in a quirky way.
