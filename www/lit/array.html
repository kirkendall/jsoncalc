Array

Array generators are represented by square brackets containing a list of values. In a JSON data file, each value must also be a literal, so the array as a whole is a literal, but JsonCalc's array generators are different. Their elements can be complex expressions, and the values of those expressions can change.

The changeable nature of JsonCalc's arrays are the reason they're called "generators" instead of "literals". Literals never change, but generators can.

Examples
[] [] This is the simplest example. An empty set of brackets returns an empty array.
[true,1,"Y",[]] [true,1,"Y",[]] Here we're generating an array with four elements. Note that the elements can be of different types.
[2+2,CURRENT_TIME] [4,"14:04:27"] Here we're generating an array with two elements, both computed. "2+2" only uses literals, but it could just as easily have accessed a variable, or extracted data from a JSON file. "CURRENT_TIME" fetches the current local time in ISO-8601 format.
Notes
  • You're allowed to place an extra comma after the last item in an array generator. This is sometimes handy if the code is machine-generated, since it means you don't need to make the last generated item be a special case. It can also save you some aggravation if you rearrange the elements manually.
    [1,2,3,] [1,2,3] Here the "3" has an extra comma after it, and that's okay.
