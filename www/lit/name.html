Names

Names (also called "keys") aren't literals. They're about as close as you can get to the opposite of a literal. They are used to access variables, or members or objects.

Names follow one of three basic formats:

  • An unquoted name must start with an underscore or letter. The rest of the name may contain any combination of letters, digits, or underscores. This is the usually most convenient format, though it's also the most constrained. Also, there's a chance that the definition of "letter" may vary from one computer to another, so I suggest you limit yourself to ASCII letters in unquoted names.
  • Backtick-quoted names. These may contain any character except another backtick. This notation is inherited from SQL.
  • String-like names, enclosed in double-quotes or single-quotes. This only works after a . operator, or before a : operator; in any other context, it'll be treated like a string literal instead of a name. Note that JSON text always double-quotes member names like this.
Examples
var a = 4;
`a` 4 "a" "a" First we define a variable named a (unquoted) and set it to 4. Then we access a (backtick-quoted) and its the same variable so we get 4. Finally, we try using double-quoted "a" but because we aren't after a . operator or before a : operator, it's just a string.
{"b":true} {"b":true} {"b":true}."b" true Here we use a double-quoted name before a colon and after a dot, and it is indeed interpreted as a name instead of a string.
