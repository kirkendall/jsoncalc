Question mark operator

bool?expr
expr : expr		Return second or third expr, depending on bool
This is an if-then-else operator. It always evaluates the first argument. If that returns a true value, then the second argument is evaluated and returned as the result of the operator; otherwise the third argument is evaluated and returned.

The :expr is optional. If you omit it, then jx will act as though you'd added :null.

Examples
true ? "Y" : "N" "Y" false ? "Y" : "N" "N" Pretty close to the simplest possible example.
x < 0 ? "negative" : x > 0 ? "positive" : "zero" "zero" This shows that ?: is right-associative, making it easy to string them together as an if-then-elseif-then-else chain.
test.scores # score < 30 ? name ["Peter","Paul","Peter","Paul"] Here's an example without the :else clause. It scans test.scores (in sampledata) for scores under 30, and returns a list of the names from those lines. Obviously it doesn't try to eliminate duplicates.
Notes
  • If you omit the :else clause anywhere except the end of the expression, then I suggest you wrap the test?then subexpression in parentheses to avoid a potential "dangling else" ambiguity.
  • JavaScript's version of ?: requires the :else clause.
  • jx, JavaScript, TypeScript, C, C++, C#, Objective C, and just about any other language that uses ?: makes it right-associative so you can easily string to together if-then-elseif-then-else expressions, as in the positive/negative/zero example above. But not all! PHP has ?: but it's left-associative, which is really hard to figure out the first time you stumble on that. The lesson: Use parentheses!
